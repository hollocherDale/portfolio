Fred Meyer | Growlerstation Management Center

A custom CMS for managing online growler menus

This application is used by all Fred Meyer stores with a growlerstation to manage their online menus. Each store has managed access to the central databse of beers so that they can easily update their menu as they change out kegs. The database is managed by Fred Meyer Beer &s; Liquor team using the admin panel of the CMS.

  • Growlers Homepage
  • Hawthorne's Store Menu
  • Store Management

I worked closely with the liquor team to build out all the features they desired include an excel importing and exporting feature. This allos them to quickly populate the databse with their existing excel documents. The design of the application was created by my corworker and supervisor, who also assisted in designing the user flow.

There is a map on the home page showing all stores grouped by state. There is a button that uses geolocation to find the closest store for to the customer.

LittleBoxes

A Gallery Showcase of Docker Containers | github

LittleBoxes is galery application for sharing Docker containers. Docker is a popular emerging virtualization technology. It is used in a variety of contexts such as deploying development environments, or deplying platforms as a service. LittleBoxes is an engaging atmosphere for discussing and sharing dockers. We integrated doge-coin tipping to connect with the positve and engaged doge coin community.

  • LittleBoxes Homepage
  • Container Page
  • User page
  • Contact Form

The user model is backed by devise and presently employs github authorization to join. This was a logical choice since the docker community is prevalent on github. Each user has a profile page for displaying their social media information as well as all containers they own. Each container has a dedicated page for showing the container and owner's information. Commenting is allowed and localized to each container page.

Dandelion

Choose your own Adventure - Crowd Sourced Story Telling | github

  • Dandelion Adventures Homepage
  • Read a book!
    It's a small book...

Dandelion, my personal project, is a portal for users to engage in choose your own adventures and then extend them as a community. Users can submit additional paths to the story that are voted on and published on a daily basis. As stories are added, the adventure becomes bigger and bigger!

Dandelion users a state machine to manage the publicatio cycle of all submissions. The top two voted submissions at each path become added in as new adventures while the remaining submissions are archived. These tasks are handled by a cron job, recurring on a daily and weekly basis.
